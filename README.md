# InexShop - Маркетплейс Игровых Предметов

Это комплексная платформа на Django для торговли внутриигровыми предметами и общения между геймерами. Платформа объединяет функционал маркетплейса, форума сообщества и социальной сети, позволяя геймерам покупать, продавать, обсуждать и обмениваться игровыми предметами в безопасной среде.

## Ключевые особенности

### Маркетплейс игровых предметов

- **Каталог игр**: Структурированный каталог поддерживаемых игр с детальной информацией о каждой.
- **Система категорий**: Тщательно организованная структура категорий для разных типов игровых предметов.
- **Объявления о продаже**: Пользователи могут размещать объявления о продаже игровых предметов с подробным описанием, ценой, качеством и другими характеристиками.
- **Фильтрация и поиск**: Продвинутая система фильтрации с гибкими параметрами, включая диапазон цен (от и до), категории предметов и характеристики.
- **Галерея изображений**: Возможность добавлять несколько изображений к каждому объявлению для лучшего представления товара.
- **Качество предметов**: Система рейтинга качества предметов (обычные, необычные, редкие, эпические, легендарные).
- **Серверы игр**: Поддержка разных серверов для каждой игры, обеспечивая точное размещение объявлений в контексте нужного игрового сервера.

### Система пользователей

- **Профили пользователей**: Детальные профили с аватарами, биографией, игровыми предпочтениями и игровой статистикой.
- **Рейтинг продавцов**: Система репутации для оценки надежности продавцов на основе истории сделок и отзывов.
- **Отслеживание активности**: Подробная статистика активности пользователя (время онлайн, количество объявлений, сделок).
- **Верификация аккаунтов**: Система подтверждения аккаунтов для повышения доверия к продавцам.
- **Безопасность аккаунтов**: Защита от взлома, блокировка при множественных неудачных попытках входа.
- **Онлайн-статус**: Отображение статуса пользователей (онлайн/оффлайн) с отслеживанием последнего времени активности.

### Система коммуникации

- **Личные сообщения**: Встроенный чат между пользователями для обсуждения сделок и других вопросов.
- **Отправка медиафайлов**: Возможность отправлять изображения, голосовые сообщения и файлы в чате.
- **Групповые беседы**: Поддержка групповых чатов для обсуждения между несколькими пользователями.
- **Уведомления**: Система уведомлений о новых сообщениях, предложениях, сделках и системных событиях.
- **Реакции на сообщения**: Возможность добавлять эмодзи-реакции к сообщениям в чате.
- **Индикаторы прочтения**: Отображение статуса прочтения сообщений.

### Игровое сообщество

- **Отзывы и рейтинги**: Система отзывов для оценки качества предложений и надежности продавцов.
- **Обсуждения игр**: Тематические разделы для обсуждения различных аспектов игр.
- **Новости и события**: Информация о новых релизах, обновлениях игр и важных событиях в игровой индустрии.
- **Обмен опытом**: Возможность делиться игровым опытом, стратегиями и достижениями.

### Административные функции

- **Панель администратора**: Расширенный интерфейс администратора для управления всеми аспектами платформы.
- **Модерация контента**: Инструменты для проверки и управления пользовательским контентом.
- **Аналитика использования**: Детальная статистика по активности пользователей, популярным категориям и объявлениям.
- **Управление пользователями**: Инструменты для управления учетными записями, разрешениями и ограничениями.

## Техническая реализация

### Архитектура

- **Фреймворк Django 5.0**: Современный Python-фреймворк для быстрой и надежной разработки веб-приложений.
- **База данных**: PostgreSQL в продакшене (через Docker), с поддержкой SQLite для разработки.
- **Docker**: Полная контейнеризация приложения для простого развертывания и масштабирования.
- **Безопасность**: Реализованы механизмы защиты от CSRF-атак, валидация всех пользовательских данных, ограничение скорости API-запросов.

### Модели данных

- **Game**: Основная модель для хранения информации об играх (название, описание, дата выпуска, разработчик, издатель).
- **GameCategory**: Модель категорий для организации игровых предметов.
- **Server**: Модель для представления игровых серверов с привязкой к конкретной игре.
- **Offer**: Центральная модель объявлений о продаже с детальной информацией о предлагаемых предметах.
- **UserProfile**: Расширенный профиль пользователя с дополнительной информацией, статистикой и настройками безопасности.
- **Conversation и Message**: Система обмена сообщениями с поддержкой вложений и медиафайлов.
- **FilterGroup, FilterOption, FilterValue**: Гибкая система фильтрации предметов с настраиваемыми опциями, диапазоном цен и дополнительными параметрами.
- **Notification**: Система уведомлений для информирования пользователей о важных событиях.

### Фронтенд

- **Адаптивный дизайн**: Полностью отзывчивый интерфейс, оптимизированный для всех устройств (десктоп, планшет, мобильный).
- **JavaScript и AJAX**: Динамическое обновление контента без перезагрузки страницы для улучшения пользовательского опыта.
- **Уведомления в реальном времени**: Мгновенные уведомления о новых сообщениях и событиях.
- **Интерактивные формы**: Удобные формы для создания и редактирования объявлений с предварительным просмотром изображений.
- **Фильтры и поиск**: Интерактивные элементы управления для настройки диапазона цен и других параметров поиска без необходимости перезагрузки страницы.

### Дополнительные функции

- **Система чата в реальном времени**: Обмен сообщениями без необходимости обновления страницы.
- **Голосовые сообщения**: Возможность записи и отправки голосовых сообщений.
- **Загрузка и обработка изображений**: Автоматическая оптимизация загружаемых изображений.
- **Отслеживание онлайн-статуса**: Автоматическое обновление статуса пользователей в реальном времени.
- **Восстановление сессий**: Сохранение состояния сессии пользователя между посещениями.

## Интерфейс пользователя

### Навигация

- **Главная страница**: Демонстрирует популярные предложения, новые игры и актуальные категории.
- **Каталог игр**: Структурированный список поддерживаемых игр с изображениями и кратким описанием.
- **Страницы игр**: Детальная информация о конкретной игре, категориях предметов и доступных серверах.
- **Страницы категорий**: Фильтрованный список предложений в конкретной категории с удобным выбором диапазона цен и других параметров.
- **Страница объявления**: Подробная информация о продаваемом предмете с фотографиями, ценой, характеристиками и возможностью связаться с продавцом.

### Профиль пользователя

- **Личный кабинет**: Управление личной информацией, объявлениями и сделками.
- **Страница профиля**: Публичный профиль с рейтингом, историей продаж и отзывами.
- **Управление объявлениями**: Интерфейс для создания, редактирования и удаления объявлений.
- **История сделок**: Отслеживание завершенных и текущих сделок.
- **Настройки безопасности**: Управление параметрами безопасности аккаунта.

### Система сообщений

- **Список бесед**: Обзор всех активных чатов с индикаторами непрочитанных сообщений.
- **Чат-интерфейс**: Удобный интерфейс для обмена сообщениями с возможностью отправки медиафайлов.
- **Уведомления**: Всплывающие и встроенные уведомления о новых сообщениях и событиях.
- **Статус сообщений**: Индикаторы доставки и прочтения сообщений.

## Настройка разработки

### Предварительные требования

- Docker и Docker Compose для контейнеризации
- Python 3.10+ (для локальной разработки)
- PostgreSQL (через Docker или локально)
- Git для управления версиями

### Docker-установка

```bash
# Клонирование репозитория
git clone <url-репозитория>
cd gameproject

# Запуск через Docker Compose
docker-compose up -d

# Миграции и начальные данные
docker-compose exec web python manage.py migrate
docker-compose exec web python manage.py loaddata initial_data

# Создание суперпользователя
docker-compose exec web python manage.py createsuperuser
```

### Локальная установка

```bash
# Клонирование репозитория
git clone <url-репозитория>
cd gameproject

# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt

# Настройка базы данных
# Отредактируйте gameproject/settings.py для локальной БД

# Миграции и начальные данные
python manage.py migrate
python manage.py loaddata initial_data

# Создание суперпользователя
python manage.py createsuperuser

# Запуск сервера разработки
python manage.py runserver
```

## Развертывание

Проект настроен для CI/CD через GitLab с автоматическим тестированием, сборкой Docker-образов и деплоем на разные окружения:

- **Тестовая среда**: Автоматический деплой при обновлении ветки develop
- **Продакшн**: Ручное подтверждение деплоя с ветки main/master

Инфраструктура продакшна включает:

- Контейнеры Docker с оркестрацией через Docker Compose
- Nginx для обслуживания статических файлов и проксирования запросов
- PostgreSQL с настроенным резервным копированием
- Redis для кэширования и управления сессиями
- Система мониторинга на базе Prometheus и Grafana
